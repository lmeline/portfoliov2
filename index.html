Projet Glosothek

Le contexte

Lors de mon stage de première année de BTS SIO, l'entreprise m'a demandé de créer un site web permettant de centraliser toutes les Google Sheets utilisées par les traducteurs pour rechercher des mots traduits. Étant donné que l’entreprise collabore avec plusieurs partenaires américains, chacun possédant son propre dictionnaire de traduction, il était essentiel de mettre en place un outil permettant de standardiser la traduction de certains mots de l’anglais vers le français.

Le besoin

L’un des principaux objectifs du projet était de regrouper tous les Google Sheets de traduction en un seul outil centralisé afin de simplifier leur accès. Grâce à cette plateforme, les traducteurs n’ont plus besoin de parcourir plusieurs fichiers pour trouver les mots traduits, ce qui leur fait gagner un temps précieux. En plus de la consultation des traductions, le site permet aux traducteurs de suggérer de nouveaux mots à la personne responsable des traductions. Afin d’assurer une gestion efficace, un traducteur administrateur a la possibilité de modifier, ajouter ou supprimer les mots proposés par les autres traducteurs.

Les outils utilisés

Pour mener à bien ce projet, j’ai utilisé VS Code comme éditeur de code. L’interface a été développée avec le framework Bootstrap, qui permet d’obtenir un design responsive et moderne. La gestion des données repose sur une base MariaDB, et le site a été conçu en PHP pour assurer son bon fonctionnement.

Syllogos

Le contexte

Dans le cadre de ma deuxième année d’alternance, on m’a demandé de créer un site permettant aux traducteurs de communiquer entre eux. Ce site ne sera pas une plateforme indépendante, mais une extension d’un éditeur de texte déjà existant. L’objectif principal de ce projet est d’aider les traducteurs à échanger lorsqu’ils ne sont pas d’accord sur une traduction écrite par un tiers. Grâce à mon site, le chapitre et le verset concernés seront directement affichés, ce qui facilitera les discussions et la vérification des traductions.

Le besoin

Ce site doit permettre aux traducteurs de discuter et justifier leurs choix de traduction en lien avec un texte précis. Il doit également s’intégrer parfaitement à l’éditeur de texte existant pour éviter toute rupture dans leur flux de travail. L’authentification des utilisateurs se fera via OAuth, en récupérant les données depuis Door43, une plateforme dédiée aux traductions collaboratives. Une fois connectés, les traducteurs pourront utiliser l’outil d’échange mis en place pour interagir efficacement.

Les outils utilisés

Pour développer ce projet, j’utilise VS Code comme environnement de développement. Le site est construit avec React pour l’interface et Express pour le backend. La base de données repose sur MongoDB, et les données sont récupérées via une API avec OAuth sur le site Door43 afin de gérer l’authentification et l’accès aux ressources.

DogsitterCRUD

Le contexte

Mon deuxième projet pour le BTS est une application CRUD destinée aux administrateurs de mon site internet de dogsitting. Cette application leur permet de gérer facilement les informations présentes sur le site, notamment en ajoutant ou en modifiant les villes, les races de chiens et les dogsitters inscrits.

Le besoin

L’objectif principal de cette application est de fournir aux administrateurs un outil efficace pour ajouter, modifier, supprimer et consulter les données du site. Grâce à cette interface, ils peuvent organiser et mettre à jour les informations sans avoir besoin d’intervenir directement dans la base de données.

Les outils utilisés

Pour développer cette application de bureau, j’utilise Visual Studio avec WPF (Windows Presentation Foundation). Ce choix permet d’offrir une interface utilisateur ergonomique et intuitive, facilitant la gestion des données du site.
